# StockIN

## 🇺🇸 English Version

## Overview
**StockIN** is a personal project: a **stock management and POS system** that handles purchases, sales, stock replenishment, and full auditing of all product movements, returns, and updates. It implements **stateless JWT authentication** and follows **Clean Code principles** and best practices for maintainable and robust architecture.

## Features
- **Stock Management** – track inventory and product levels.  
- **Point of Sale (POS)** – handle sales with automatic stock deduction.  
- **Purchases and Replenishment** – add products and update stock.  
- **Auditing** – track all operations including sales, purchases, returns, and product edits.  
- **Authentication** – JWT-based stateless security.  
- **Read-only Transaction Logs** – GET endpoints for auditing and historical queries.  
- **Clean Code & Architecture** – modular, maintainable, and robust design.

## Tech Stack
- **Java 21**  
- **Spring Boot 3.5.4**  
- Web, Security, Data JPA, Validation 
- **Databases** – H2 (runtime, for testing) and PostgreSQL (runtime, production)  
- **JWT Authentication**
- **Swagger/OpenAPI** – API documentation  
- **Lombok** – for reducing boilerplate code  
- **Build Tool** – Maven  

## API Documentation
The API is fully documented with **Swagger/OpenAPI**

## 🇧🇷 Versão em Português

## Visão Geral
**StockIN** é um projeto pessoal: um **sistema de gerenciamento de estoque com PDV** que gerencia compras, vendas, reposição de estoque e auditoria completa de todas as movimentações de produtos, devoluções e alterações. Implementa **autenticação stateless via JWT** e segue os **princípios de Clean Code** e boas práticas de arquitetura para garantir manutenção e robustez.

## Funcionalidades
- **Gerenciamento de Estoque** – controle de inventário e níveis de produtos.  
- **Ponto de Venda (PDV)** – realização de vendas com atualização automática do estoque.  
- **Compras e Reposição** – cadastro de produtos e atualização de estoque.  
- **Auditoria** – acompanhamento de todas as operações, incluindo vendas, compras, devoluções e alterações de produtos.  
- **Autenticação** – segurança stateless baseada em JWT.  
- **Logs de Transações Somente-Leitura** – endpoints GET para auditoria e consultas históricas.  
- **Clean Code & Arquitetura** – design modular, manutenível e robusto.

## Tech Stack
- **Java 21**  
- **Spring Boot 3.5.4**  
- Web, Security, Data JPA, Validation  
- **Bancos de Dados** – H2 (runtime, para testes) e PostgreSQL (runtime, produção)  
- **Autenticação JWT**  
- **Swagger/OpenAPI** – documentação da API  
- **Lombok** – redução de código boilerplate  
- **Build Tool** – Maven  

## Documentação da API
A API está totalmente documentada com **Swagger/OpenAPI**.
